## 진단 퀴즈 오답 (1회차 → 정정 완료)

### 🟢 Q2 — CNCF 성숙도: Graduated 아닌 것 찾기
- 카테고리: CNCF 생태계·거버넌스
- 1차 답: c (Envoy) ❌ → 정정: **d (KEDA)** ✅
- 핵심 정리: Envoy는 Graduated. KEDA는 상대적으로 신생·이벤트 기반 오토스케일러로 아직 Graduated 아님.
- 연속정답: 1 / 복습 예정: D+3

### 🟢 Q6 — 오토스케일링 짝 중 "잘못된" 것
- 카테고리: 아키텍처·원칙
- 1차 답: b ❌ → 정정: **d** ✅
- 핵심 정리: VPA 설명(b)은 사실로 맞음. KEDA는 노드 CPU 클럭이 아니라 **이벤트 기반으로 워크로드(Pod) 수**를 조정 → d가 틀린 설명.
- 연속정답: 1 / 복습 예정: D+3

### 🟢 Q7 — 관측성 3대 축
- 카테고리: 관측성(Observability)
- 1차 답: c ❌ → 정정: **a (Metrics, Logs, Traces)** ✅
- 핵심 정리: CPU/Memory/Disk는 관측 대상 자원 지표일 뿐, 3대 축은 메트릭·로그·트레이스.
- 연속정답: 1 / 복습 예정: D+3

### 🟢 Q9 — 분산 트레이싱 프로젝트 조합
- 카테고리: 관측성(Observability)
- 1차 답: a (Grafana+Loki=시각화+로그) ❌ → 정정: **c (Jaeger + OpenTelemetry)** ✅
- 핵심 정리: Jaeger=트레이싱 백엔드, OTel=트레이스/메트릭/로그 계측 표준.
- 연속정답: 1 / 복습 예정: D+3
