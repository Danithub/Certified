## A. CNCF 생태계 · 거버넌스 (집중 영역)

**Q1.** CNCF 프로젝트 성숙도 단계를 낮은 순 → 높은 순으로 올바르게 나열한 것은?
- a) Incubating → Sandbox → Graduated
- b) Sandbox → Incubating → Graduated
- c) Sandbox → Graduated → Incubating
- d) Experimental → Incubating → Stable

**내 답:** b

**Q2.** 다음 중 **Graduated** 프로젝트가 아닌 것은? (2026 기준 일반적 분류)
- a) Kubernetes
- b) Prometheus
- c) Envoy
- d) KEDA

**내 답:** c

**Q3.** CNCF가 소속된 상위 재단은?
- a) Apache Software Foundation
- b) Linux Foundation
- c) OpenStack Foundation
- d) Eclipse Foundation

**내 답:** b

## B. Cloud Native 아키텍처 · 원칙

**Q4.** "Cloud Native"의 핵심 특성으로 가장 거리가 먼 것은?
- a) 탄력적 확장성(elastic scalability)
- b) 느슨한 결합(loose coupling)
- c) 수동 스케일 조정 우선(manual-first scaling)
- d) 복원력(resilience)

**내 답:** c

**Q5.** 서버리스(Serverless)에 대한 설명으로 옳은 것은?
- a) 서버가 물리적으로 존재하지 않는다
- b) 개발자가 서버 프로비저닝·관리를 신경 쓰지 않고 코드 실행에 집중한다
- c) 항상 컨테이너보다 느리다
- d) 오토스케일이 불가능하다

**내 답:** b

**Q6.** 오토스케일링 짝으로 잘못된 것은?
- a) HPA — Pod 개수를 수평 확장
- b) VPA — Pod의 리소스 요청/제한을 조정
- c) Cluster Autoscaler — 노드 수를 조정
- d) KEDA — 노드의 CPU 클럭을 조정

**내 답:** b

## C. 관측성 (Observability, 신버전 Architecture 소속)

**Q7.** 관측성의 3대 축(three pillars)은?
- a) Metrics, Logs, Traces
- b) Alerts, Dashboards, Reports
- c) CPU, Memory, Disk
- d) Ingress, Egress, Internal

**내 답:** c

**Q8.** Prometheus의 메트릭 수집 방식은 기본적으로?
- a) Push (에이전트가 서버로 전송)
- b) Pull (서버가 타깃의 /metrics를 스크레이프)
- c) Streaming (지속적 소켓)
- d) Batch (일 1회)

**내 답:** b

**Q9.** 분산 트레이싱과 가장 관련 깊은 프로젝트 조합은?
- a) Grafana + Loki
- b) Fluentd + Elasticsearch
- c) Jaeger + OpenTelemetry
- d) Harbor + Notary

**내 답:** a

## D. App Delivery · GitOps (집중 영역)

**Q10.** GitOps의 핵심 원칙으로 옳은 것은?
- a) 운영자가 kubectl로 직접 수동 변경하는 것을 권장
- b) Git 저장소를 시스템의 원하는 상태(desired state)에 대한 단일 진실 공급원(source of truth)으로 사용
- c) CI 파이프라인만 있으면 GitOps다
- d) 롤백은 불가능하다

**내 답:** b

**Q11.** Helm에서 "Chart"는 무엇인가?
- a) 실행 중인 Pod의 로그
- b) K8s 리소스를 템플릿화해 패키징한 배포 단위
- c) 모니터링 대시보드
- d) 네트워크 정책의 종류

**내 답:** b

## E. Container Orchestration (가볍게 확인)

**Q12.** CRI(Container Runtime Interface)를 구현하는 런타임은?
- a) etcd
- b) containerd / CRI-O
- c) CoreDNS
- d) Calico
